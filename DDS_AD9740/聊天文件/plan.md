# 项目背景
我们正在开发一个基于EP4CE10 FPGA + STM32H723 + AD9740 DAC的DDS任意波形发生器。FPGA与STM32通过16位异步FMC总线通信，FPGA内部实现DDS核心，驱动DAC输出。

# 已完成工作（请勿重复设计）
- **STM32端**：FMC接口初始化（Bank1，16位SRAM模式），已测试对FPGA寄存器的读写功能。CubeMX工程已配置USART1（115200 8N1）作为调试接口。
- **FPGA端**：PLL（50MHz→125MHz）配置完成；FMC从机接口（`stm32_fmc_16bit.v`）实现，包含地址译码和寄存器读写逻辑；双端口波形RAM（1024×10位）已例化。
- **文档**：寄存器映射（地址偏移0x0000~0x040C）、影子寄存器机制、DDS参数（32位累加器，FTW公式）均已明确。

# 待实现功能（按优先级）
## 高优先级（核心DDS通路）
1. **FPGA DDS核心模块**  
   - 32位相位累加器（带FTW输入和相位复位）  
   - 波形RAM读地址生成（取累加器高10位）  
   - 输出数据寄存，与125MHz DACCLK对齐（建议使用ODDR原语）  
   - 实现影子寄存器加载逻辑（FTW_Shadow → FTW_Active，由更新使能触发）
2. **STM32波形生成与加载**  
   - 生成1024点正弦波查找表（10位，范围0~1023）  
   - 批量写入波形RAM（FMC连续写0x0000~0x03FF）  
   - 实现FTW计算函数（根据目标频率，32位无符号整数）  
   - 实现原子更新流程（先写FTW低16位→高16位→写更新使能）
3. **DAC输出控制**  
   - 实现DAC使能/禁用寄存器（0x040C bit0），禁用时强制输出中间值（0x200）

## 中优先级（调试与用户交互）
4. **USART命令解释器**  
   - 接收命令（如查询寄存器状态、DAC输出状态、波形校验）  
   - 通过USART1发送响应（使用中断或轮询）
5. **波形校验机制**  
   - 校验写入RAM的数据是否完整（可回读比对或计算校验和）

# 设计约束与注意事项
- FPGA时钟域：所有DDS逻辑需在125MHz下工作，FMC接口信号需经两级同步处理。
- 影子寄存器：必须确保FTW只有在更新使能后才生效，避免部分更新导致的频率毛刺。
- FMC时序：STM32配置为`AddressSetupTime=4, DataSetupTime=3, AddressHoldTime=15`，FPGA需满足相应建立/保持。
- MPU配置：STM32对FMC区域（0x60000000）需设为非缓存、非缓冲。
- 波形RAM写安全：在批量写入新波形前，建议先禁用DAC输出，防止读出旧数据造成输出混乱。
