本项目基于已有的 **AD9740 DDS 任意波形发生器** 工程（FPGA 顶层 `FMC_Demo`，STM32 侧 `dds.c/h`），扩展 **AD9220 ADC 高速数据采集** 功能。所有新增模块必须与现有 DDS 模块**并行独立**，共享 FMC 总线但使用不同的地址偏移。请按以下阶段逐步给出完整实现。
ADCdata为12bit

#### 【阶段 0】地址空间规划（避免与现有 DDS 冲突）
现有 DDS 占用地址偏移（STM32 视角，16 位字地址）：
- `0x0000~0x03FF`：波形 RAM（1024×16）
- `0x0400`：FTW 低 16 位（只写）
- `0x0401`：FTW 高 16 位（只写）
- `0x0404`：更新触发（只写，写 1 装载 FTW）
- `0x0408`：相位复位（只写，自清零）
- `0x040C`：DAC 控制（bit0 = 使能）

请为 **ADC 采集子系统** 分配新的地址偏移（建议从 `0x1000` 开始），至少包含：
- **ADC 控制寄存器**（启用/关闭，bit0）
- **采样率步进值低 16 位**（只写）
- **采样率步进值高 16 位**（只写）
- **步进更新触发**（写 1 装载完整 32 步进值，遵循“先写低→再写高→最后写触发”的原子模式）
- **FIFO 数据端口**（只读，读取时自动弹出下一个 16 位数据）
- **FIFO 状态寄存器**（只读，含空标志、剩余字数等，可选）

要求：所有新地址偏移必须与现有 DDS 地址**不重叠**，并列出映射表。

#### 【阶段 1】FPGA 新增 ADC 时钟与复位域
现有设计使用 **125 MHz** 作为主逻辑时钟（FMC 及 DDS）。ADC 需要独立的 **10 MHz** 采样时钟（由 PLL 的另一个输出提供）。
- 修改 FPGA 顶层 `FMC_Demo.v`，在 `pll1` 中增加 `c2` 输出 = 10 MHz（0°相移），专供 ADC 采样时钟。
- 将 10 MHz 时钟引出到 FPGA 引脚 `ADC_CLK_OUT`（连接 AD9220 CLK）。
- 为 ADC 域增加独立的复位同步器（基于 10 MHz 和全局 `sys_rst_n`），确保 ADC 相关逻辑在 PLL 锁定后稳定复位。
- 注意：ADC 域的时钟与 FMC 域的 125 MHz 为**异步时钟**，后续需在时序约束中设置异步组。

---

#### 【阶段 2】FPGA 新增 FMC 地址译码与影子寄存器（ADC 部分）
在现有的 `stm32_fmc_16bit.v` 模块中，扩展地址译码逻辑，识别上述 ADC 新地址偏移。
- 实现 **两个 16 位影子寄存器**（`adc_step_low`, `adc_step_high`）用于保存 STM32 写入的步进值。
- 实现 **步进更新触发**：当 STM32 写入触发地址（且写数据为 1）时，将 `{adc_step_high, adc_step_low}` 组合成 32 位 `adc_step_full`，并同步到 ADC 时钟域（使用两级同步器或异步寄存器，因为该值变化极慢，可采用握手机制或直接使用双寄存器同步）。
- 实现 **ADC 控制寄存器**（使能位），同样需同步到 ADC 域。
- 实现 **FIFO 读数据通道**：当 STM32 读 `FIFO_DATA` 地址时，从异步 FIFO 的读端口取数，并自动产生读请求（单周期脉冲），同时返回 FIFO 输出数据。
- **注意**：所有新增的 FMC 访问逻辑必须与现有的 DDS 写/读操作互不影响。

#### 【阶段 3】FPGA 新增 32 位 DDS 累加器（ADC 采样率控制）
参照 DDS 波表生成的相位累加器思想，但用于控制 ADC 数据抽取。
- 编写新模块 `adc_decimator`（或集成到单独文件），使用 **10 MHz ADC 时钟**。
- 内部包含 **32 位相位累加器**，每个周期累加 `adc_step_full`（来自同步后的步进值）。
- 采用 `(next_phase < phase_acc)` 检测无符号溢出，产生单周期 `write_pulse`。
- 在溢出时锁存当前 ADC 输入数据（`ADC_D[11:0]`，FPGA 引脚输入），并补为 16 位（高 6 位补 0）作为 FIFO 写数据。
- 当 ADC 控制寄存器使能位为 0 时，强制累加器停止（或输出固定值），且不产生写脉冲。

#### 【阶段 4】FPGA 异步 FIFO 例化与连接
使用 Quartus IP 核（`dcfifo`）或手工推断双时钟双口 RAM 实现异步 FIFO。
- 配置：宽度 16 位，深度 1024，写时钟 = 10 MHz（ADC 域），读时钟 = 125 MHz（FMC 域）。
- 写端：写使能 = `write_pulse`（来自累加器溢出），写数据 = 补位后的 ADC 数据。
- 读端：读使能 = 由 FMC 读请求产生（仅当 FIFO 非空时），读数据输出至 FMC 数据总线。
- 将 FIFO 的 `wrusedw` 或 `rdusedw` 连接到状态寄存器（可选），供 STM32 查询。
- 注意：FIFO 的复位必须同时复位两个时钟域，使用异步复位或同步复位到各自时钟。

#### 【阶段 7】STM32 驱动层新增 ADC 操作函数（参照 `dds.c` 风格）
在 `STM32/Core/Src/` 下新建 `adc.c` 和 `adc.h`，实现：
- `ADC_Init()`：配置 FMC 映射地址宏。
- `ADC_Enable(bool enable)`：写控制寄存器。
- `ADC_SetSampleRate(float freq_hz)`：计算 32 步进值（使用 `double`），按“写低 16 → 写高 16 → 写触发”顺序原子写入（参考 `DDS_SetFrequency` 的实现，含关中断和内存屏障）。
- `ADC_GetFIFOData()`：直接从地址读取 16 位数据（用于非 DMA 调试）。
- `ADC_GetFIFOStatus()`：读取状态寄存器（空标志/剩余字数）。

注意：所有地址偏移使用宏定义，避免硬编码。

#### 【阶段 8】STM32 DMA 双缓冲读取 FIFO（乒乓模式）（后面所有暂时不用实现，这一步阶段8告诉我如何使用cubemx配置DMA）
配置 DMA（如 DMA2，数据流 0）为 **外设到内存** 模式，外设地址 = FIFO 数据地址，内存地址为双缓冲数组。
- 使用 **双缓冲模式**（`HAL_DMAEx_MultiBufferStart`）或 **半传输中断 + 全传输中断** 模拟乒乓。
- 缓冲区大小建议 256 或 512 个 16 位字，根据 FIFO 深度 1K 调整。
- 开启 DMA 传输完成中断和半传输中断。
- 在中断回调中：
  - 处理已填满的半区（调用用户回调 `ADC_ProcessData` 传入缓冲区指针和长度）。
  - 切换缓冲区指针（双缓冲模式硬件自动切换，只需获取当前缓冲区地址）。
- 注意：由于 FMC 区域是 Device 类型，DMA 需配置为 **非缓存** 访问，且需在每次切换缓冲区后调用 `SCB_CleanInvalidateDCache` 或使用 `MPU` 配置该区域为 **共享设备** 以确保数据一致性。

- 提供常见问题排查表：
  - FIFO 空读导致数据错误 → 检查 DMA 启动时机，确保 FIFO 非空。

  - 数据错位（高/低字节颠倒）→ 检查 FMC 数据宽度和 STM32 端字节序。
  - DMA 无法启动 → 检查 MPU 配置、DMA 优先级、外设地址对齐。
